This document describes the lexicography module of the Lexicon Model for Ontologies (lemon) as a result of the work of the Ontology Lexicon community group (OntoLex). The module is targeted at the representation of dictionaries and any other linguistic resource containing lexicographic data, and addresses structures and annotations commonly found in lexicography. This module operates in combination with the lemon core module, referred to as OntoLex.

Introduction

Background and motivation

The lemon model provides a core vocabulary (OntoLex) to represent linguistic information associated to ontology and vocabulary elements. The model follows the principle of semantics by reference in the sense that the semantics of a lexical entry is expressed by reference to an individual, class or property defined in the ontology.

lemon as the outcome of the OntoLex group and its previous version (lemon [1]) have been increasingly used in the context of dictionaries and lexicographical data to convert existent lexicographic information into the standards and formats of the Semantic Web. Such preliminary experiences comprise monolingual [2], bilingual [3], and multilingual [4] dictionaries, as well as diachronic [5], dialectal [6], and etymological ones [7], among others. The added value of using linked data technologies in lexicography and its implications for the micro and macro structure of dictionaries have been explored as well by several authors [8, 9]. After analysing the literature, the proposers of this module perceived an obvious need for reaching some agreement that allows for a better and more interoperable migration of existing dictionaries into linked data [10]. For illustration, the OGL ontology [11] has its own notion of dictionary entry materialised in the ogl:Entry class, while in [4] the ad-hoc kd:dictionaryEntry relation was introduced in the conversion of the KD Multilingual Global Series dictionaries, i.e, different researchers introduced their own modelling solutions to account for similar notions. Being interoperability a key issue in linked data technologies, building a common space in which these concepts can be agreed on and commonly defined comes as a logical step. The OntoLex community is the natural forum to accomplish this for several reasons:

  1. The extended use of lemon to convert language resources (including lexicographical resources) into linked data,
  2. the nature of the lemon model being descriptive but not prescriptive, which facilities neutrality towards different lexicographic views,
  3. the coming together of the lexicography and the Semantic Web communities, and
  4. the possibility of reusing already available mechanisms in lemon (instead of "re-inventing the wheel").

Aim and scope

The main goal of this module is to complement the lemon core module, OntoLex, and to overcome its limitations when modelling lexicographic information as linked data in a way that is agnostic of the underlying lexicographic view and minimizes information loss.

The scope of the model is two-fold:

  1. modelling existing lexicographic resources as linked data, to allow their later publication and interoperation by following Semantic Web standards, and
  2. providing a conceptual / abstract model of language and linguistic objects in lexicography that will allow to build linked data-based lexicographic resources from scratch.
In fact, although the primary goal might be to model existing dictionaries as linked data, this can be done by extracting the features they have in common and modelling in a way that is decoupled from specific linguistic views. Thus, this module should remain highly re-usable across dictionaries, as well as useful for new linked data-based lexicographic resources built from scratch (see the notion of linked data-native dictionaries at [12]). We assume of course that many specific ingredients from specific dictionaries and views will still need ad-hoc constructs, but having a few common entities defined in the module will help to overcome the limitations that have been detected when current Ontolex is applied to lexicographic data, and can greatly benefit interoperability.

In terms of applying the module, we propose the following best practice or "rule of thumb" when representing a dictionary as linked data: always that the entities in the Ontoex model suffice for representing the resource's lexicographic information (e.g., lexical entry, part of speech, translation, ...), then the Ontolex lexicographic module should not be instantiated; in case that some lexicographic information cannot be modelled by using neither the core of Ontolex nor its other modules (e.g., to denote sense ordering), then the module will be instantiated, but avoiding duplicities and keeping extra information to the minimum.

Lexicographic resource

There are certain situations (see [10]) in which there is no one to one mapping between the entries in a dictionary or lexical resource and the lexical entries in an Ontolex lexicon. For instance, synonyms and translations of some dictionary entries can appear as "extra" information of such entries, without their own entry, in a source dictionary. However, in Ontolex, such synonyms and translations should be treated as "first class citizens" with their own lexical entries and senses. In such situations, there is a disparity between the created Ontolex lexicon and the original dictionay. In order to support the original grouping of entries in a source dictionary or, more in general, to represent a particular view or organisation of the lexicographic information in a language resource (which might differ from its arrangement in an Ontolex lexicon) the class lexicog:LexicographicResource is introduced.
